# Debugger 5 分钟接线指南

本文面向首次接线 `RDC-Agent-Frameworks/debugger` 的维护者或高级用户，目标是在 5 分钟内完成一条可验证的手工接线。

## 1. 选择宿主模板

从 `debugger/platforms/<platform>/` 中选择目标宿主模板，例如 `codex`、`claude-code`、`cursor`。

## 2. 覆盖共享正文

将 `debugger/common/` 整体覆盖到目标模板根目录的 `common/`。

## 3. 显式绑定 `Tools`

编辑：

```text
common/config/platform_adapter.json
```

将：

```json
"tools_root": "__CONFIGURE_TOOLS_ROOT__"
```

替换为有效的 `RDC-Agent-Tools` 根目录。

## 4. 运行接线校验

在模板根目录执行：

```bat
python common/config/validate_binding.py --strict
```

通过标准：

- `tools_root` 不是占位值
- `validation.required_paths` 在 `tools_root` 下全部存在
- `tool_catalog.snapshot.json` 与 `Tools` 的 catalog 对齐
- 宿主入口文件和共享文档入口都存在

## 5. 再进入宿主

接线通过后，再在目标宿主中打开模板根目录，并按 `debugger/README.md` 的流程提供 `.rdc` 后开始调试。

## 6. 当前口径

- `debugger` 是当前唯一进入首发 GA 规划的 framework。
- `analyzer` / `optimizer` 仍是 `incubating`。
- 当前 `Tools` snapshot 应对齐 `202` 个 tools，并包含新增的只读 `rd.vfs.*` 探索接口。
