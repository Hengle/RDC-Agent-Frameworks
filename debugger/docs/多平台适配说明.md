# RenderDoc/RDC GPU Debug · 多平台适配说明

本文说明 `RenderDoc/RDC GPU Debug` 如何在不同宿主平台下落地，同时保持 `common/` 中的 SSOT 不漂移。

## 路径约定

除非特别说明，以下路径均相对于 `debugger/`。

## 核心原则

- `common/agents/` 是角色职责与 Prompt 的 SSOT。
- 平台能力层的实现默认值集中在 `common/config/platform_adapter.json`。
- 平台适配物只能绑定宿主所需配置，不得把具体实现名升级为框架概念。
- 用户要求 `CLI` 模式时，平台侧应先引导 Agent 阅读 `docs/cli-mode-reference.md`。

## 适配层职责

### Claude Code

- 使用 `.claude/settings.json` 挂接质量门槛。
- 读取 `common/AGENT_CORE.md` 作为框架核心说明。

### Code Buddy

- 通过 `skills/debugger/SKILL.md` 加载框架知识。
- 通过 `.mcp.json` 绑定具体平台 `MCP` 实现。
- `CLI` 模式下必须先读 `skills/debugger/references/cli-mode-reference.md`。

### Claude Work / Copilot / Manus

- 这些适配物应保留角色镜像与最少接入说明。
- 若宿主缺少强制 Hook，则由 Prompt 明确质量门槛和模式边界。

## 维护命令

```bash
python debugger/scripts/validate_tool_contract.py --strict
python debugger/scripts/sync_platform_agents.py
```
